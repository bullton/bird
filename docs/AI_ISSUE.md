# AI 物种信息获取的已知问题

## 问题描述

当前 AI 获取物种信息（目/科/属/学名等）依赖 MiniMax-M3 纯文本模型，存在以下问题：

1. **学名错误**：AI 在将中文名推断为拉丁学名时容易产生错误（如把"斑皇鸠"推断为不存在的 `Ducula bracteata`，实际正确学名是 `Ducula bicolor`）
2. **无实时数据**：AI 仅使用训练数据，无法联网校验，无法获取最新分类信息
3. **相似物种混淆**：中文名相似或英文名相似的物种容易被混淆

## 原因

- 项目不支持 MCP（Model Context Protocol），无法调用联网搜索工具
- AI 只能通过 prompt 描述来"推断"物种信息，而非查询权威数据库
- 鸟类学名在不同资料库中可能存在差异，AI 知识库可能过时或不准确

## 可能的解决方案

### 方案 A：本地鸟类学名数据库（推荐）

**原理**：维护一份权威鸟类学名映射表（中文名 → 学名/科/目等），直接查表不依赖 AI

**优点**：
- 数据 100% 准确，来源于 Avibase、eBird 等权威数据库
- 响应速度快，不依赖外部 API
- 一次维护，长期使用

**缺点**：
- 需要人工维护数据（但鸟类数据相对稳定，更新频率低）
- 无法处理未知物种

**实现方式**：
1. 从 Avibase 或 eBird 导出鸟类数据，生成 `birds.json`
2. AI 识别出中文名后，先查本地数据库获取学名
3. 描述文本（habitat/diet/description）仍由 AI 生成

---

### 方案 B：调用权威鸟类 API

**原理**：AI 识别出中文名后，调用第三方 API 获取标准学名

**可用 API**：
1. **Avibase API** (`https://api.avibase.nl/api/v2/`) - 全球鸟类分类学数据库
2. **iNaturalist API** (`https://api.inaturalist.org/v1/`) - 观测数据，支持中文名搜索
3. **eBird API** (`https://api.ebird.org/`) - 需要申请 key，但数据权威

**优点**：
- 数据权威，实时更新
- 不需要本地维护数据

**缺点**：
- 依赖外部服务可用性
- 可能需要 API Key
- 网络延迟

---

### 方案 C：使用有联网能力的 AI 模型

**原理**：切换到支持 MCP 或工具调用的 AI（如 Claude 3.5/GPT-4o）

**MCP 工具**：
- `孔雀mcp` - 联网搜索工具
- `gitlab-mcp` - 搜索工具
- 可以让 AI 实时查询 Avibase、eBird 等

**优点**：
- AI 自主判断，流程最简单
- 可以处理边缘物种

**缺点**：
- MiniMax 目前不支持 MCP
- 需要切换到付费模型（如 Claude/GPT）

---

### 方案 D：MiniMax VL 模型（图像识别）

**原理**：MiniMax 可能有多模态模型（MiniMax-VL），用于图像识别可能更准确

**当前**：
- 项目使用 MiniMax-M3 纯文本模型做识别
- 图片识别准确率取决于 AI "脑补" 能力

**改进**：
- 如果 MiniMax 有 VL 模型，可以上传图片直接识别
- 图片识别 + 本地数据库查表组合使用

---

### 方案 E：混合方案（AI + 人工校验）

**原理**：AI 生成的数据需要用户确认后才能入库

**流程**：
1. AI 返回识别结果（中文名/学名/描述）
2. 用户在「待确认」列表中查看
3. 用户确认/修正后数据正式入库
4. 学名会自动与本地数据库校验，错误时提示修正

**优点**：
- 最大限度减少错误数据
- 用户参与度高

**缺点**：
- 增加用户操作步骤

---

## 推荐实施路径

**短期（立即实施）**：
1. 方案 A - 建立本地鸟类数据库，中文名查表获取标准学名
2. 方案 E - 增加 AI 识别后的用户确认流程

**中期**：
3. 方案 B - 接入 iNaturalist/Avibase API 作为备份数据源

**长期**：
4. 方案 C - 考虑切换到支持 MCP 的 AI 模型

## 当前临时处理方式

- 在 SpeciesDetail 页面开放「学名」编辑框，用户可手动修正 AI 返回的错误学名
- 「修正」按钮会重新调用 AI，但仍依赖 AI 的知识库质量

## MiniMax CLI 调研

MiniMax 官方提供 CLI 工具 (`minimax-cli`)，可以：
- 调用 MiniMax API
- 支持多模态输入

**服务器上检查结果**：未安装 MiniMax CLI

**如果安装**：
```bash
npm install -g @minimax-api/minimax-cli
minimax configure --api-key YOUR_KEY
```

但 CLI 本质上仍是调用 API，无法解决知识库准确性问题。
